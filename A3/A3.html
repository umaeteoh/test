Task 3

Name Web Attacks
Cross-Site Scripting



Source: https://www.cloudflare.com/en-gb/learning/security/threats/cross-site-scripting/ 		The attacker sends a link of a website with malicious Javascript code to the victim. When the victim clicks on the link, the victim's browser launches the website and executes the malicious script. The attacker gains access to sensitive data and install malware.

https://www.cloudflare.com/en-gb/learning/security/threats/cross-site-scripting/
SQL Injection



Source: https://www.cloudflare.com/en-gb/learning/security/threats/sql-injection/ 		The attacker injects a malicious SQL query into the input fields of an application. This manipulates the application to run the query on the victim's database. The attacker is able to obtain sensitive information, modify data or execute commands on the database.

https://owasp.org/www-community/attacks/SQL_Injection
Broken Authentication



Source: https://www.prplbx.com/resources/blog/broken-authentication/ 		Weaknesses in the authentication or session management of an application create vulnerabilities for attackers to exploit. When users are not properly authenticated, attackers are able to obtain stolen login credentials or hijack session IDs. This leads to data breaches and unauthorised access to sensitive information.

https://www.prplbx.com/resources/blog/broken-authentication/
Drive-by Downloads



Source: https://us.norton.com/blog/malware/what-are-drive-by-downloads 		Malware is automatically downloaded onto the user's device after the user visits a website. Attackers are able to hijack the user's device, destroy data or obtain personal information. This attack takes advantage of security flaws in not up to date software.

https://www.kaspersky.com/resource-center/definitions/drive-by-download
Password-based Attacks



Source: https://us.norton.com/blog/emerging-threats/password-attack 		Category of attacks that involve cybercriminals using tools to crack users' passwords. Such methods involve brute force, phishing, keyloggers and credential stuffing. Successful password cracking allows attackers to gain unauthorised access and impersonate victims.

https://www.educative.io/answers/what-is-a-password-attack
Distributed Denial of Service Attacks



Source: https://www.cloudflare.com/en-gb/learning/ddos/what-is-a-ddos-attack/ 		The attacker disrupts the functionality of a network or server by overwhelming the infrastructure with a flood of traffic. This results in normal traffic and legitimate users being denied access to the network. This is commonly utilised for hacktivism, revenge or blackmail.

https://www.cloudflare.com/en-gb/learning/ddos/what-is-a-ddos-attack/
Man in the Middle Attacks



Source: https://www.wallarm.com/what/what-is-mitm-man-in-the-middle-attack 		The attacker stealthily inserts themselves between communications between two parties and intercepts the information transferred. The information is manipulated without the knowledge of the two parties involved. This allows the attacker to obtain and exploit sensitive personal data from the unsuspecting parties.

https://en.wikipedia.org/wiki/Man-in-the-middle_attack
Ransomware



Source: https://blog.ariacybersecurity.com/blog/just-what-is-a-ransomware-attack-and-can-you-prevent-one 		The victim mistakenly downloads the ransomware from clicking a link, giving the attacker access to their files. The attacker encrypts the files to make them inaccessible to the victim. Then, the attacker demands payment in exchange for the decryption key or to prevent leaks of the victim's data.

https://en.wikipedia.org/wiki/Ransomware
Trojan



Source: https://vpnoverview.com/internet-safety/malware/trojan-horse/ 		Malware that is disguised as legitimate software to trick users into downloading it. Once installed, Trojans are able to obtain sensitive data, install additional malware and gain control of the victim's device. This attack can be prevented by installing new software updates and not clicking on unfamiliar links.

https://au.norton.com/blog/malware/what-is-a-trojan
Directory Traversal



Source: https://www.wallarm.com/what/path-traversal-attack 		The attacker is able to exploit software on a web server to gain access to directories outside the web root folder. This enables unauthorised access to configuration/ system files and the ability to execute commands on the server. This can be prevented with validation of the user's input.

https://owasp.org/www-community/attacks/Path_Traversal
